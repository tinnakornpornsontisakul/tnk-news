# SKILL — TokenX Weekly Roundup (คู่มือรวบรวมข่าวรายสัปดาห์)

คู่มือนี้ใช้ผลิต **จดหมายข่าวสรุปรายสัปดาห์ของ TokenX** ด้าน Real World Asset (RWA),
Tokenization, Blockchain, Fintech และข่าวตลาดทุนไทย (SET) — เป็นเอกสาร 2 ภาษา (ไทย/อังกฤษ)
ที่สลับธีมสว่าง–มืดได้ และพิมพ์เป็น PDF ได้

---

## 1. ไฟล์ในระบบ (Files)

| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | ดีไซน์หลัก / หน้าเว็บ (โครงสำเร็จ) — ไม่ต้องแก้ · เปิดไฟล์นี้เพื่อดูผล |
| `news-data.js` | **ตัวรวมไฟล์รายสัปดาห์ (loader)** — มี manifest `WEEK_FILES` รวมทุกไฟล์ใน `weeks/` |
| `weeks/YYYY-MM-DD.js` | **1 ไฟล์ = 1 สัปดาห์** — ข้อมูลข่าวจริงของแต่ละฉบับ |
| `weeks/_TEMPLATE.js` | โครงข้อมูลเปล่า (placeholder) — คัดลอกไปเริ่มสัปดาห์ใหม่ |
| `assets/` | โลโก้ TokenX (สว่าง/มืด) และไอคอน + ภาพพื้นหลัง |

**กฎเหล็ก:** แก้เฉพาะไฟล์ใน `weeks/` และ manifest `WEEK_FILES` ใน `news-data.js` เท่านั้น
ห้ามแก้โครงดีไซน์ `index.html` (กล่องไฮไลต์ data/know/takeaway/quote ฝังอยู่ในไฟล์นี้แล้ว) เว้นแต่ต้องการเปลี่ยนรูปแบบจริง ๆ

---

## 1.1 โมเดลข้อมูล: 1 สัปดาห์ = 1 ไฟล์ (Weekly archive)

ข้อมูลแต่ละสัปดาห์อยู่ **คนละไฟล์** ในโฟลเดอร์ `weeks/` — ตั้งชื่อไฟล์ตามวันที่ของฉบับ `weeks/YYYY-MM-DD.js`

แต่ละไฟล์ลงทะเบียนตัวเองผ่าน `registerWeek({ ... })`:

```js
// weeks/2026-06-18.js
registerWeek({
  meta: { issue: 42, date: "2026-06-18", dateEN: "Jun 12–18, 2026", dateTH: "12–18 มิ.ย. 2026" },
  market: { growth: {...}, tokens: [...] },
  lead: {...},
  stories: [...],
  briefs: [...],
});
```

`news-data.js` (loader) จะโหลดทุกไฟล์ใน `WEEK_FILES` → ประกอบเป็น
`window.RWA_NEWS_ARCHIVE` (**เรียงใหม่→เก่าตาม `meta.date` อัตโนมัติ** และกันวันที่ซ้ำ) และตั้ง `window.RWA_NEWS` = ฉบับล่าสุด

### ➕ เพิ่มสัปดาห์ใหม่ (3 ขั้น)
1. คัดลอก `weeks/_TEMPLATE.js` → `weeks/YYYY-MM-DD.js` (ใช้วันที่ของฉบับเป็นชื่อไฟล์)
2. กรอกข่าวของสัปดาห์นั้น — อย่าลืมใส่ `meta.date` (ISO `YYYY-MM-DD`) ให้ตรงกับชื่อไฟล์
3. เพิ่มชื่อไฟล์ 1 บรรทัดใน `WEEK_FILES` ของ `news-data.js`

> **ไม่ต้องแก้ไฟล์เก่าเลย** — แต่ละสัปดาห์แยกกันสมบูรณ์ จึงไม่มีความเสี่ยงกระทบของเก่า จะเก็บถาวรง่าย
> จะ“ซ่อน”ฉบับเก่าชั่วคราวก็แค่นำชื่อออกจาก `WEEK_FILES` (ไฟล์ยังอยู่) — ใส่กลับได้ภายหลัง

> ปัจจุบันใน `weeks/` มีไฟล์ตัวอย่าง `2026-06-18` (ฉบับจริง) + `2026-06-11`, `2026-06-04`
> ที่เป็น**ฉบับตัวอย่าง** (โครงซ้ำฉบับล่าสุด) — แทนที่ด้วยข่าวจริงของแต่ละสัปดาห์

## 2. ขั้นตอนต่อสัปดาห์ (Weekly workflow)

1. **สร้างไฟล์สัปดาห์** — copy `weeks/_TEMPLATE.js` → `weeks/YYYY-MM-DD.js` แล้วเพิ่มชื่อไฟล์ใน `WEEK_FILES` ของ `news-data.js`
2. **กำหนดช่วงเวลา** — ข่าวย้อนหลัง **7 วันล่าสุด** เท่านั้น ใส่ `meta.issue`, `meta.date` (ISO) และวันที่ทั้ง 2 ภาษา
3. **รวบรวมข่าว** ตามหมวด (ดูข้อ 3–4) แล้วคัดให้เหลือข่าวที่ "ขยับตลาด" จริง
4. **เขียน 2 ภาษา** ตามกฎข้อ 6 + ตารางคำทับศัพท์ข้อ 7
5. **เลือก 1 ข่าวเด่น (lead)** ของฉบับ ใส่ในฟิลด์ `lead`
6. **ตรวจลิงก์อ้างอิงทุกข่าว** ต้องเปิดได้จริง
7. **เปิด `index.html`** ตรวจผล สลับธีม/ภาษา/ฉบับ แล้ว export PDF ถ้าต้องการ

## 3. หมวดข่าว (Topics) และปริมาณ

`topic` ที่ใช้ได้: `rwa` · `tokenization` · `blockchain` · `fintech` · `set`

- ตั้งเป้า **2–4 ข่าวต่อหมวด** + ข่าวเด่น 1 (lead, อยู่ในหมวด rwa)
- ถ้าสัปดาห์ไหนหมวดใดไม่มีข่าวเด่นพอ ให้ลดจำนวน — **คุณภาพ > ปริมาณ** อย่าใส่ข่าวน้ำท่วมทุ่ง
- หมวด **SET** รับเฉพาะข่าว **สินทรัพย์ดิจิทัล / Digital Asset ของตลาดทุนไทย**
  (SET, SEC, BOT) — ไม่ใช่ข่าวหุ้นทั่วไป
  - **แหล่งหลักของหมวดนี้:** SET News & Alerts → `https://www.set.or.th/th/market/news-and-alert/news`
    เน้นดึงจากหน้านี้ และ **เลือกเฉพาะรายการที่เกี่ยวกับ Digital Asset / สินทรัพย์ดิจิทัล** (รายละเอียดแหล่งข่าว ดูข้อ 4)
- เติม **ข่าวสั้น "โดยสรุป" (`briefs`) 3–5 รายการ** สำหรับข่าวรองที่ไม่ถึงขั้นมีเนื้อเต็ม

## 4. แหล่งข่าวแนะนำ (Sources)

### ⚠️ กฎเหล็ก: ใช้เฉพาะแหล่งที่ "ปลอดภัย + น่าเชื่อถือ" เท่านั้น

ข่าวในทุกหมวด **ต้องนำมาจากแหล่งที่ปลอดภัยและเชื่อถือได้เท่านั้น** เรียงตามลำดับความน่าเชื่อถือ (เลือกอันดับสูงสุดที่หาได้):

1. **แหล่งต้นทาง / ทางการ (Primary / official) — ดีที่สุด:** ประกาศของหน่วยงานกำกับ (SEC, BOT, SET, ก.ล.ต., MAS), เอกสารยื่นทางการ (filings / prospectus), ข่าวประชาสัมพันธ์ทางการบนเว็บไซต์ของบริษัทเอง, บล็อก/เอกสารทางการของโปรเจกต์
2. **สำนักข่าวการเงินชั้นนำที่มีกองบรรณาธิการ:** Reuters, Bloomberg, Financial Times, CoinDesk, The Block, Blockworks
3. **แพลตฟอร์มข้อมูล on-chain ที่ได้รับการยอมรับ** (ใช้สำหรับ "ตัวเลข/มูลค่า" ไม่ใช่ความเห็น): RWA.xyz, DeFiLlama, Messari, Dune, Token Terminal

**❌ ห้ามใช้เด็ดขาด (unsafe / unreliable):**
- **โซเชียล/ข่าวลือ:** โพสต์ X / Telegram / Discord ที่ไม่มีแหล่งยืนยัน, "alpha" จากบัญชีนิรนาม
- **เว็บที่เชื่อถือไม่ได้:** บล็อกนิรนาม, คอนเทนต์โฆษณา/PR ที่จ่ายเงิน (sponsored), aggregator ที่ไม่ระบุที่มา, เนื้อหา AI ที่ไม่ได้ตรวจสอบ
- **เว็บอันตราย:** ลิงก์ phishing/หลอกลวง, เว็บปลอมเลียนแบบแบรนด์, หน้าที่ขอ connect wallet / seed phrase

**ก่อนใส่ทุกข่าว ต้องผ่านข้อนี้ (ข้อแรกสำคัญสุด):**
- [ ] **⚠️ เปิดอ่านลิงก์จริงทุกอัน แล้วยืนยันว่า "หัวข้อ + เนื้อข่าวที่เขียน ตรงกับเนื้อหาในลิงก์นั้นจริง"** — อย่าเชื่อแค่ชื่อบทความ/snippet จากผลค้นหา · รายละเอียดทุกอย่าง (ตัวเลข, ชื่อบริษัท/บุคคล, วันที่) ต้อง **มีอยู่ในแหล่งที่อ้างนั้นเอง** ห้ามหยิบรายละเอียดจากบทความอื่นมาปนในลิงก์เดียว
- [ ] **ยืนยัน ≥2 แหล่ง** สำหรับข่าวที่มีผลต่อตลาด (cross-check) — ถ้ายืนยันได้แหล่งเดียว ให้ระบุชัดว่าเป็น "รายงาน/ยังไม่ยืนยัน"
- [ ] **`url` ชี้โดเมนทางการจริง** (เช่น `set.or.th`, `sec.or.th`, `reuters.com`, `coindesk.com`) เปิดได้จริง — ไม่ใช่ลิงก์ย่อ/redirect ที่ตรวจสอบไม่ได้
- [ ] **`source` ใช้ชื่อจริง** ที่ตรงกับ `url` — ห้ามแต่งชื่อให้ดูน่าเชื่อถือเกินจริง

### แหล่งแนะนำตามหมวด (ทุกอันผ่านเกณฑ์ด้านบนแล้ว)

- **RWA / Tokenization:** RWA.xyz, DeFiLlama, Blockworks, CoinDesk, The Block, Securitize, Ondo
- **Blockchain:** Messari, The Block, official chain blogs (Ethereum, Solana), Chainlink
- **Fintech:** Reuters, Bloomberg, Financial Times, TechCrunch, Fintech Times, ผู้ให้บริการ (Visa, Stripe, PayPal)
- **SET / ไทย (หมวด `set`):**
  - **แหล่งหลัก — สแกนหน้านี้ก่อนเสมอ:** SET News & Alerts → `https://www.set.or.th/th/market/news-and-alert/news`
  - **คัดเฉพาะข่าวที่เกี่ยวกับ Digital Asset / สินทรัพย์ดิจิทัลเท่านั้น** — ข้ามข่าวหุ้น / บริษัทจดทะเบียน / ผลประกอบการทั่วไปทั้งหมด
  - เสริมด้วยประกาศทางการเมื่อเกี่ยวข้อง: ก.ล.ต. `sec.or.th`, ธปท. `bot.or.th`

> หมายเหตุ: อ้างอิงจากแหล่งต้นทาง/ทางการเป็นหลัก ใส่ `source` เป็นชื่อสำนักข่าว และ `url` เป็นลิงก์จริง

### ⏱️ กฎความสดของข่าว (Recency) — สำคัญมาก

ข่าวที่ดึงมา **ต้องเป็นข่าว "สด" ภายใน ~7 วันล่าสุด** (อิงช่วงวันที่ของฉบับ) — ปัญหาที่พบบ่อยคือ search ดึงบทความเก่าข้ามเดือน/ข้ามปีปนมา ป้องกันแบบนี้:

- **ใส่กรอบเวลาในคำค้นทุกครั้ง** เช่น เดือน+ปีปัจจุบัน, "this week", "past 7 days", "latest" — และมองหาตัวเลือกเรียงผลแบบ "ล่าสุดก่อน"
- **เปิดดูวันที่เผยแพร่ของทุกบทความก่อนใช้** ถ้าเก่ากว่า ~2 สัปดาห์ หรือหาวันที่ชัดเจนไม่ได้ → **ทิ้ง**
- ยึด **วันที่ของ "เหตุการณ์" ในข่าว** ไม่ใช่วันที่หน้าเว็บ (บางหน้าอัปเดตของเก่าแต่เปลี่ยนวันที่)
- **ข้าม** บทความสรุปย้อนหลัง / ภาพรวมรายปี / price prediction / explainer (เช่น "2026 review", "market report") — ใช้เฉพาะข่าว **"เหตุการณ์ที่เพิ่งเกิด"**
- หาข่าวสดของหมวดไหนไม่พอ → **ลดจำนวนข่าวหมวดนั้น** ดีกว่าเอาข่าวเก่ามาเติม (ความสด > ปริมาณ)
- ตรวจสุดท้าย: ทุกข่าวในฉบับควรอยู่ในกรอบ `meta.date` ± ไม่กี่วัน

## 5. เกณฑ์คัดข่าว (Selection criteria)

เลือกข่าวที่ตอบได้ว่า **"ทำไมผู้อ่านสายการเงิน/สถาบันต้องสนใจ"**:

- มีผลต่อตลาด เม็ดเงิน หรือกฎเกณฑ์ (institutional inflows, ใบอนุญาต, มาตรฐานใหม่)
- เป็นก้าวสำคัญเชิงโครงสร้าง (ขึ้น production, ลดค่าธรรมเนียม, ดีลแรก/รายแรก)
- มีตัวเลขหรือข้อเท็จจริงยืนยันได้

**หลีกเลี่ยง:** ข่าวลือ ราคาเหรียญรายวัน โฆษณา/PR ล้วน ๆ และ "data slop"
(ตัวเลข/ไอคอนที่ไม่ได้ให้ข้อมูลจริง)

## 6. การเขียน 2 ภาษา (Bilingual writing rules)

แต่ละข่าวมีทั้ง `en` และ `th` โครงเหมือนกัน (ดู schema ใน `weeks/_TEMPLATE.js`)

**โครงสร้าง title** = array 3 ช่อง `[ก่อน, คำที่เน้นสี, หลัง]`
ใส่วลีสำคัญ 2–4 คำไว้ช่องกลางเพื่อให้ขึ้นสีไฮไลต์ เช่น
`["JPMorgan นำร่อง ", "การชำระราคาบนเชน", " ให้ลูกค้าสถาบัน"]`

**ความยาว:**
- พาดหัว (title): สั้น กระชับ < ~12 คำ
- เนื้อข่าว (body): 2–3 ประโยค (ข่าวเด่น lead ใส่ได้ 2 ย่อหน้า)
- ไฮไลต์ note: 1 ประโยค

**โทน:** กระชับ เป็นกลาง แบบ desk จดหมายข่าวการเงิน ไม่โฆษณา ไม่ตื่นเต้นเกินจริง

**ภาษาไทย — ใช้คำทับศัพท์ภาษาอังกฤษกับศัพท์เทคนิค** (ดูตารางข้อ 7)
เขียนไทยปนอังกฤษให้อ่านลื่นแบบที่คนสายฟินเทคไทยใช้จริง เช่น
"เม็ดเงินสถาบันไหลเข้าสู่ Real World Asset แบบ Token ทำสถิติใหม่"

## 7. ตารางคำทับศัพท์ (Transliteration glossary)

ศัพท์เทคนิคต่อไปนี้ **ให้คงเป็นภาษาอังกฤษ** ในข้อความภาษาไทย (อย่าแปลเป็นไทย):

| ความหมาย | ใช้ในภาษาไทยว่า |
|---|---|
| สินทรัพย์ในโลกจริง | **Real World Asset** (ย่อ **RWA**) |
| โทเคน / การแปลงเป็นโทเคน | **Token** / **Tokenization** / "แบบ Token" / "Tokenized" |
| บล็อกเชน | **Blockchain** |
| เทคโนโลยีการเงิน | **Fintech** |
| เหรียญคงที่ | **Stablecoin** |
| สินทรัพย์ดิจิทัล | **Digital Asset** |
| การเงินไร้ตัวกลาง | **DeFi** |
| เลเยอร์สอง / โรลอัป | **Layer-2** / **rollup** |
| สัญญาอัจฉริยะ | **smart contract** |
| มูลค่าทรัพย์สินสุทธิ | **NAV** |
| ผลตอบแทน | **yield** |
| สภาพคล่อง | **liquidity** หรือ "สภาพคล่อง" ได้ทั้งคู่ |
| เชนแบบมีสิทธิ์เข้าถึง | **permissioned chain** |
| การทำงานร่วมข้ามเชน | **interoperability** / "ข้ามเชน" |
| ผู้ดูแลทรัพย์สิน | **custody** / "การเก็บรักษาทรัพย์สิน" |

**คงเป็นไทย** (อ่านลื่นกว่า): การชำระราคา/ชำระเสร็จ, พันธบัตร, สินเชื่อ, หลักประกัน,
การกำกับดูแล, ผู้ลงทุน/นักลงทุน, ตลาดหลักทรัพย์, "บนเชน" (on-chain)

**ชื่อหน่วยงานไทย:** ใช้ตัวย่อภาษาอังกฤษ — ตลาดหลักทรัพย์ฯ (**SET**), สำนักงาน ก.ล.ต. (**SEC**),
ธนาคารแห่งประเทศไทย (**BOT**) — เขียนเป็นตัวย่ออังกฤษได้เลยในเนื้อข่าวและใน `source` เช่น "SEC", "BOT", "SET"

**ชื่อบริษัท/แพลตฟอร์ม:** คงภาษาอังกฤษเสมอ (JPMorgan, Stripe, Visa, Ethereum, Solana, ฯลฯ)

## 8. กล่องไฮไลต์ (Highlight box) — ปรับได้หลายแบบ

กล่องไฮไลต์ใต้แต่ละข่าวยืดหยุ่นได้ เลือก `hlType` ให้เหมาะกับเรื่อง — ใช้เท่าที่ช่วยเล่าเรื่อง อย่าใส่ทุกข่าว:

- **`"data"`** → กล่องตัวเลขเด่น · `highlight: { label, value, note }`
  ใช้เมื่อมีตัวเลขชัด เช่น `value: "$2.1B"`, `"T+0"`, `"−60%"`, `"฿2,000M"`
- **`"know"`** → กล่อง "รู้ไว้ / Know this" (ไอคอนหลอดไฟ) · `highlight: { label, note }`
  ใช้อธิบายศัพท์/แนวคิดให้คนทั่วไปเข้าใจ (เช่น fractionalization, tokenized deposit)
- **`"takeaway"`** → กล่องสรุปประเด็นสำคัญ (ไอคอนดาว) · `highlight: { label, note }`
  ใช้ตอกย้ำ "สิ่งที่ต้องจำ" ของข่าว — `note` เขียนเป็นประโยคสรุปหนัก ๆ 1 ประโยค
- **`"quote"`** → คำพูด / Pull quote · `highlight: { note, by }`
  `note` = ข้อความคำพูด, `by` = ชื่อ–ตำแหน่งผู้พูด (ต้องเป็นคำพูดจริงจากแหล่งข่าว)
- **ไม่ใส่ hlType** → ข่าวไม่มีกล่องไฮไลต์ (ปกติ)

### 8.1 ให้ AI ตัดสินใจเอง — วิเคราะห์เนื้อข่าวแล้วเลือกรูปแบบ

**อย่ากำหนด hlType ตายตัว** ให้ AI อ่านและวิเคราะห์เนื้อข่าวแต่ละชิ้น แล้ว *เลือกเอง* ว่าจะนำเสนอแบบไหนให้สื่อสารได้ดีที่สุด ตามธรรมชาติของเนื้อหา:

| ลักษณะเนื้อข่าว | เลือกใช้ |
|---|---|
| มีตัวเลข/สถิติเด่น **ชิ้นเดียว** ที่สรุปข่าวได้ ($, %, เท่า, T+0) | `data` |
| มีศัพท์/กลไกที่คนทั่วไปอาจไม่เข้าใจ ต้องอธิบาย | `know` |
| เป็นข้อสรุปหรือนัยเชิงกลยุทธ์ที่อยาก "ตอกย้ำให้จำ" | `takeaway` |
| มีคำพูดผู้บริหาร/ผู้กำกับดูแลที่ทรงพลังและมีแหล่งจริง | `quote` |
| ข่าวตรงไปตรงมา ไม่มีจุดที่ต้องเน้น | ไม่ใส่ |

**หลักการตัดสินใจ:**
- ดูว่า "แก่นของข่าว" คืออะไร — ตัวเลข, แนวคิด, ข้อสรุป, หรือเสียงของคน — แล้วจับคู่กับรูปแบบข้างต้น
- **เลือกหนึ่งแบบต่อข่าว** และทำให้ทั้งฉบับ *หลากหลาย* — อย่าใช้ `data` ซ้ำทุกชิ้น ให้รูปแบบสะท้อนเนื้อหาจริง ไม่ใช่ใส่ตามสูตร
- ใส่ไฮไลต์เฉพาะเมื่อช่วยเล่าเรื่อง ถ้าไม่จำเป็นให้เว้นว่าง

`label` มาตรฐาน: data → "ตัวเลขสำคัญ / By the numbers" หรือ "ทำไมจึงสำคัญ / Why it matters";
know → "รู้ไว้ / Know this"; takeaway → "สรุปสำคัญ / Key takeaway"

> กล่องทุกแบบใช้สีของหมวดข่าวเป็นเส้นขอบ/ไอคอนโดยอัตโนมัติ และปรับตามธีมสว่าง–มืดเอง
> เวลา export PDF กล่องไฮไลต์ทุกแบบพับเป็นคอลัมน์เดียวให้พอดีหน้า

## 8C. แดชบอร์ดภาพรวม (เมื่อเลือกตัวกรอง "ทั้งหมด")

แต่ละไฟล์สัปดาห์มีฟิลด์ `market` ของตัวเอง — เมื่อผู้อ่านเลือก "ทั้งหมด" ส่วนบนสุดจะแสดงแดชบอร์ดของ**ฉบับที่กำลังเปิดอยู่** (เปลี่ยนตามฉบับ/วันที่ที่เลือกจากปฏิทิน):
- **สถิติข่าว** — นับจำนวนข่าว / หมวด / แหล่งข่าว ให้เองจากเนื้อหา ไม่ต้องกรอก
- **กราฟการเติบโต Tokenized RWA `market.growth`** — `label` (2 ภาษา), `value` (มูลค่ารวมบนเชน), `delta` (% เปลี่ยนแปลง), `up` (true=ขึ้น), และ `points` (ชุดข้อมูลแนวโน้ม ~8–12 จุด) วาดเป็นกราฟพื้นที่
- **ราคาเหรียญ RWA `market.tokens`** — แสดงเป็น **แถบราคาไหลต่อเนื่อง (ticker tape) ปักอยู่บนสุดของหน้า** (เลื่อนหน้าลงก็ยังติดอยู่ด้านบน · แสดงทุกตัวกรอง ไม่ใช่เฉพาะ "ทั้งหมด") แต่ละเหรียญใส่ `sym`, `name`, `price`, `chg`, `up`, และ `bar` (0–100 ความยาวแท่งเชิงเปรียบเทียบ) — ดู **รายชื่อ 20 เหรียญมาตรฐานในข้อ 8C.1**

ค่าเหล่านี้เป็นภาพรวมตลาดของสัปดาห์นั้น (ไม่ใช่ข่าวเดี่ยว) — **อัปเดตตัวเลขในไฟล์สัปดาห์ทุกฉบับ**
ถ้าสัปดาห์ไหนไม่ใส่ `market` แดชบอร์ดจะแสดงเฉพาะสถิติข่าว ส่วนแถบราคาจะซ่อนอัตโนมัติ

### 8C.1 รายชื่อเหรียญ RWA แนะนำ (20 ตัวดัง — ชุดมาตรฐานของ `market.tokens`)

**ค) Token ของแพลตฟอร์ม/โปรโตคอล RWA** (ราคาผันผวนได้มาก)

## 9. แสดงผล & ส่งออก (Render & export)

- เปิด `index.html` — ค่าเริ่มต้นเป็น **ธีม Navy มืด + ภาษาไทย + ฉบับล่าสุด**
- ปุ่มมุมขวาบน: สลับธีมสว่าง–มืด (ดวงอาทิตย์/ดวงจันทร์) และเลือกภาษา (ไทย/English)
- **เลือกฉบับ/สัปดาห์** (หัวกระดาษด้านขวา):
  - ปุ่ม **‹ ›** เลื่อนไปฉบับเก่ากว่า/ใหม่กว่าทีละฉบับ (ปิดอัตโนมัติเมื่อสุดทาง)
  - กดที่ป้าย **"ฉบับที่ … "** (ไอคอนปฏิทิน) → เปิด **ปฏิทิน** วันที่ที่มีข้อมูล (`meta.date`) จะไฮไลต์และกดได้ วันอื่นกดไม่ได้ เลือกแล้วดึงข้อมูลฉบับนั้นมาแสดง
- **ตัวกรองหัวข้อ** (แท็บใต้หัวเรื่อง): แท็บแบบมีเส้นใต้สีประจำหมวด เลือกดูเฉพาะหมวดได้ (ข่าวสั้น "โดยสรุป" + แดชบอร์ดแสดงเฉพาะตอนเลือก "ทั้งหมด")
- พิมพ์/บันทึก PDF ได้ทันที (เลย์เอาต์ปรับเป็นสีอ่อนสำหรับงานพิมพ์อัตโนมัติ — พิมพ์เฉพาะฉบับที่เปิดอยู่)
